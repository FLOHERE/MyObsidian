- 명령어 : /speckit.constitution 
	- 목적 : 프로젝트에게 일종의 규칙 부여.(기술스택, 코딩 컨벤션, 아키텍처 원칙 등)
	- 산출물 : constitution.md
		- 메모리라는 폴더 안에 생기는데, 너무 많은 프롬프트가 쌓이면 가끔 변하게 되는 경우가 있음. 그렇기 떄문에 md 파일 형태로 미리 규칙을 백업 시키는것.

```md
1. 모든 산출물과 대화는 한글로 진행한다.
2. 테서트 중심 개발(TDD)을 원칙으로 한다.
3. 사용자의 개인정보 및 민감한 데이터를 다루는 만큼. 보안을 최우선으로 한다.
4. 항상 공식 문서를 바탕으로 개발한다.
5. 응집도는 높게, 결합도는 낮게 개발한다.
```

- 명령어 : /speckit.specify
	- 요구사항 정의 및 구체화
	- 목적 : 구현하고자 하는 기능의 '무엇' 과 '왜'에 집중
	- 산출물 : spec.md

```md
1. 이 서비스는 취업정보 사이트 입니다. 사용자가 웹 브라우저로 접근하여 현재 채용이 진행중인 회사의 정보를 확인할 수 있어야 한다.

2. 제공되는 정보
	- 회사명
	-  회사 위치
	- 채용 분야
	- 채용 조건
	- 지원 기간
	- 지원 방법
	- 채용 정보가 있는 url을 등록 할 수도 있다.
	- 이메일 사용 -> 정보 제공 가능
		- 해당 이메일은 제목이 반드시 "연암 취업정보"로 시작해야 함.
		- 이메일에는 url 포함 가능
		- url을 오로지 관리자 권한을 가진 사람만 접근 가능해야 한다.
```
