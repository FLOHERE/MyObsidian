# 임베디드 시스템
 : 특정한 기능이나 목적을 수행하기 위해 하드웨어 + 소프트웨어가 결합된 전용 컴퓨팅 시스템

# 임베디드 프로그래밍의 특징
1. 실행되는 기계가 아닌 다른 기계에서 개발된다.
2. 부팅에서부터 응용프로그래밍 실행까지 하나의 덩어리 이다.
3. 자원 사용에 제약이 많다.
	1. 플래시 용량이 충분 X
	2. 메모리 용량이 충분 X
	3. 프로세서 성능 충분 X
	4. 전원 배터리 용량 충분 X
	   => 최적화를 위해 하드웨어를 직접 다룰 필요가 있음
4. 임베디드 시스템은 하드웨어 이해 필요
	1. 메모리 맵
	2. 입출력 연결
5. 인공지능의 도움을 받기 쉽지 않다.(현재는...)
	1. 마이크로 컨트롤러의 도움을 받기 쉽지 않음
	2. 응용에 맞게 맞춤화 되는 경우가 많다.
	3. 정보가 공개되지 않는다.
	4. 명세서, 회로도, 사용 시나리오 입력시 될지도...

# 실습 장치
- NXP사의 S32K312EVB-Q172 - 자동차용 임베디드 시스템
- 참고자료
	- 사용자 설명서
	- 회로도
	- 배치도
	- S32K3XX 자동차용 마이크로컨트롤러 
		- 데이터 시트
		- 래퍼런스 메뉴얼
	- S32 Design Studio 및 추가 패키지(IDE)
		- 설명서

# 프로그램 개발 환경
- 교재
	- 크로스 컴파일러 : 윈도우에서 만든 파일이 다른 하드웨어 운영체제에서도 파일이 실행될 수 있도록 하는 컴파일러
		1. arm-none-ebai-gcc
		2. arm-none-ebai-ld
	- makefile : 컴파일 과정 자동화 설정 파일
- S32 Design Studio : 개발도구, gcc, gdb 컴파일러와 디버거가 있다.

# 임베디드 프로그래밍 방법
- 기계어 / 어셈블리어로 작성
- C언어
	- 포인터 사용 -> 주소 직접 접근
	- 디바이스 드라이버 사용 -> API 함수 호출
	- 운영체제를 사용 -> 시스템 콜 호출
- 일반 응용 프로그램 방식
	- 리눅스 or 윈도우 용 프로그램 작성
