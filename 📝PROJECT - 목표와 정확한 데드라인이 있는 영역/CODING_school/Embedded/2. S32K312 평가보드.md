# S32K12 마이크로 컨트롤러

## 구성요소
1. 코어 프로세서(Arm Processor 사용) : 프로그램의 명령 해석 -> 실행하는 컴퓨터의 전체 동작을 처리하는 중앙처리 장치
	- 코어 : 칩 안에 들어있는 명령어를 해석하고 실행하는 중심 연산 장치
2. 메모리
3. 저장장치(HDD)
4. 입출력 핀
5. UART : 직렬통신에서 데이터를 비동기적으로 송수신 하기 위해 사용되는 하드웨어 모듈 OR 프로토콜
6. I2C(Inter - Integragted Circuit) : MCU가 여러 센서나 주변 장치와 통신하기 위한 
   양방향 직렬버스로, 한쌍의 선으로 여러 장치와 통신가능
7. SPI(Serial Periphera Interface) : 클록 신호 기반 마스터와 하나 이상의 슬레이브(주변 장치)간 데이터를 동기식 직렬방식으로 고속 전송하기 위한 풀 듀렉스 통신 프로토콜
	1. 풀 듀렉스 : 동시에 통신 주고받기
8. CAN
9. ADC : 아날로그 신호 -> 디지털 신호로 바꾸는 역할
10. PWM(Pulse width Modulation - 펄스 폭 변조) : 디지털 신호의 펄스 폭(HIGH 지속 시간) 조절 
    -> 아날로그 신호처럼 조작 가능


>[!note] UART 추가 설명
>MCU와 센서가 데이터를 서로 주고 받고, MCU가 PC와 데이터를 주고 받는데, 이런 연결에서 문자나 숫자 데이터를 한 비트씩 순서대로 주고받는 통신 장치다.
>- RX, TX로 1:1 통신을 함.

# GPIO(범용 I/O)
- 마이크로 컨트롤러의 핀 용도 중 하나
- 마이크로 컨트롤러의 핀은 한개에 여러 용도 사용 가능
- GPIO로 설정된 핀은 출력, 입력, 입출력 중 하나로 추가 설정된다.
- GPIO로 설정된 핀의 값은 1비트이다.
- GPIO로 설정된 핀은 특정 주소를 통해 접근된다.

```assembly
LDR    r0, = 0x00000064.   ; r0 <- 즉시값 0x64 (주소 100) 로드
MOVS r1, #1                      ;r1 <- 1 (저장할 값)
STR    r1, [r0]                     ; 메모리[0x64] <- r1 (word 단위 저장)
```

>[!note] 릴리즈
>필요한 코드만 올리는것.
