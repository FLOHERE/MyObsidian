
# 자바 프그래밍 범위
## 1. 기본 구문과 구조
- 변수, 데이터 타입, 연산자
```jsx
A) 자바에서 변수는 하나의 데이터를 저장하는 메모리 공간이며 데이터 타입에는 int, char, String, boolean, float, double, long 등이 있고 연산자에는 +, -, *, /, %가 있으며 예를 들어 int a = 10 + 2; 와 같이 사용한다.
```

- 조건문(if, switch)
```jsx
A) 자바에서 조건문은 조건식의 참과 거짓에 따라 실행 흐름을 제어하는 구조로 if 문은 조건이 참일 때 실행되고 
switch 문은 변수의 값에 따라 분기되며 예를 들어 if(score >= 60) 또는 switch(menu) 와 같이 사용한다.
```

- 반복문(for, while, do-while)
```jsx
A) 자바에서 반복문은 특정 코드를 반복 실행하기 위한 구조로 for 문은 반복 횟수가 명확할 때, 
while 문은 조건이 참인 동안, do-while 문은 최소 한 번 실행되며 예를 들어 for(int i=0;i<5;i++) 와 같이 사용한다.
```

## 2. 객체지향 프로그래밍(OOP)
- 클래스와 객체의 정의 및 사용
```jsx
A) 자바에서 클래스는 객체를 생성하기 위한 설계도이고 객체는 클래스를 기반으로 생성된 인스턴스로 
예를 들어 class Car { } 와 Car c = new Car(); 와 같이 사용한다.
```

- 상속 : 클래스간의 상속을 이해하고 super, this 키워드 사용법
```jsx
A) 자바에서 상속은 기존 클래스의 기능을 재사용하는 것으로 extends 키워드를 사용하며 
super는 부모 클래스의 멤버를 참조하고 
this는 현재 객체를 가리키며 예를 들어 super(); 
또는 this.name = name; 과 같이 사용한다.
```

- 다형성 : 오버라이딩과 오버로딩의 차이 및 적용
```jsx
A) 자바에서 다형성은 하나의 메서드가 여러 형태로 동작하는 성질로 
오버라이딩은 부모 메서드를 재정의하는 것이고 
오버로딩은 매개변수를 달리하여 같은 이름의 메서드를 정의하는 것이다.
```

- 캡슐화 : 데이터 은닉과 접근 제어자(private, public, protected)
```jsx
A) 자바에서 캡슐화는 객체의 데이터를 외부로부터 숨기는 것으로 
private, public, protected 접근 제어자를 사용하며 
- private는 해당 클래스 내부에서만 접근 가능하며, 외부 클래스에서는 접근할 수 없다.
- public은 모든 클래스에서 접근 가능한 접근 제어자
- protected는 같은 패키지나 상속 관계에 있는 자식 클래스만 접근 가능하다.
```

- 추상화 : 추상 클래스와 인터페이스 구현의 차이점
```jsx
A)
추상화란 여러 객체들의 공통된 특징을 추출하여 하나의 상위 개념으로 정의하고, 
불필요한 세부 구현을 숨김으로써 프로그램의 복잡성을 줄이고 코드의 재사용성과 유지보수성을 향상시키는 것이다.

 자바에서 추상 클래스는 abstract 키워드를 사용하여 일부 메서드를 구현할 수 있고 
인터페이스는 모든 메서드가 추상 메서드이며 
implements 키워드로 구현되고 다중 구현이 가능하다.
```

## 3. 예외처리
- try-catch-fanially 구조의 이해와 적용
```jsx
A) 자바에서 try-catch-finally 구조는 예외 처리를 위해 사용되며 
try는 예외 발생 코드, catch는 예외 처리, finally는 항상 실행되는 블록으로 
예를 들어 try { } catch(Exception e) { } finally { } 와 같이 사용한다.
```

- 사용자 정의 예외의 생성과 예외 던지기
```jsx
A) 자바에서 사용자 정의 예외는 Exception 클래스를 상속하여 생성하며 
throw 키워드를 사용해 예외를 발생시키고 throws 키워드로 예외를 전달할 수 있다.
```

## 4. 자료 구조 활용
- 배열, ArrayList, HashMap
```jsx
A) 자바에서 배열은 동일한 타입의 데이터를 연속적으로 저장하고 
ArrayList는 크기가 가변적인 리스트이며 
HashMap은 키와 값을 쌍으로 저장하는 자료구조로 
예를 들어 int[] arr, new ArrayList<>(), new HashMap<>() 과 같이 사용한다.
```

- 스택, 큐, 링크드 리스트, 이진트리
```jsx
A) 자바에서 스택은 후입선출(LIFO)로 단방향 입출력 구조이며, 한번에 하나의 데이터만 넣고 뺄 수 있다.
스택의 선언은 `Stack<T> 스택 이름 = new Stack<>();` 형태로 선언할 수 있으며,
데이터 타입은 클래스 또는 래퍼 클래스로 선언할 수 있다.add와 push로 값을 넣고 뺌

큐는 선입선출(FIFO)로 프런트로 데이터를 삭제 하고, 리어로 데이터를 삽입하는 구조로 
`Queue<Integer> queue = new LinkedList<>();` 형태로 사용한다.

링크드 리스트는 노드가 서로 주소 포인터를 가르키며 참조하는 형태.
`import java.util.LinkedList;` 로 import를 선언 후
`LinkedList<Integer> list = new LinkedList<>();`로 사용

이진트리는 각 노드가 최대 두 개의 자식을 가지는 트리 구조이다. 자바에서는 왼쪽, 오른쪽 노를 각자 지정해서 코드를 작성한다.

class Node {
    int data; // 노드에 저장될 데이터
    Node left; // 왼쪽 자식 노드
    Node right; // 오른쪽 자식 노드
		
    public Node(int data) {
        this.data = data;
        this.left = null;
        this.right = null;
    }
}
```

# 알고리즘 범위
## 1. 알고리즘 개념
- 알고리즘의 정의와 중요성
```jsx
A) 알고리즘은 문제를 해결하기 위한 유한한 단계들의 절차이며 효율적인 알고리즘은 시간과 자원을 절약하여 프로그램의 성능을 향상시킨다.
```

- 알고리즘 표기법 : 빅오, 빅 오메가, 빅 세타 표기법
```jsx
A) 빅오 표기법은 알고리즘의 최악 시간 복잡도, 빅 오메가는 최선 시간 복잡도, 빅 세타는 평균 또는 정확한 시간 복잡도의 범위를 나타내는 표기법이다.
```

- 시간 복잡도와 공간 복잡도
```jsx
A) 시간 복잡도는 알고리즘 실행에 필요한 시간의 증가량을 나타내고 공간 복잡도는 실행 중 필요한 메모리 사용량을 나타낸다.
```

```jsx
시간복잡도 단계 (빠른순으로)  
**[fast] O(1) < O( log n ) < O(n) < O(n log n) < O(n^2)**   
  
**O(1) – 상수 시간** : 문제를 해결하는데 오직 한 단계만 처리함.  
**O(log n) – 로그 시간** : 문제를 해결하는데 필요한 단계들이 연산마다 특정 요인에 의해 줄어듬.  
**O(n) – 직선적 시간** : 문제를 해결하기 위한 단계의 수와 입력값 n이 1:1 관계를 가짐.  
**O(n log n) - 선형로그형:**  문제를 해결하기 위한 단계의 수가 N*(log2N) 번만큼의 수행시간을 가진다.  
**O(n^2) – 2차 시간 :** 문제를 해결하기 위한 단계의 수는 입력값 n의 제곱.
```

## 2. 정렬 알고리즘
- 버블 정렬, 선택 정렬, 삽입 정렬
```jsx
A) 버블 정렬은 인접한 두 원소를 비교하여 교환을 반복함으로써 정렬하는 알고리즘이다.

A) 선택 정렬은 전체 데이터 중 최소값을 선택하여 앞쪽부터 차례로 정렬하는 알고리즘이다.
->교환 횟수 최소 / 임베디드 / 소규모 데이터

A) 삽입 정렬은 데이터를 하나씩 선택하여 이미 정렬된 부분에 적절한 위치에 삽입하는 알고리즘이다.
-> 거의 정렬됨 / 실시간 추가 / 소규모 데이터
```

- 병합 정렬, 퀵 정렬, 힙 정렬
```jsx
A) 병합 정렬은 데이터를 분할한 후 각각 정렬하고 다시 병합하는 분할 정복 기반의 정렬 알고리즘이다.
->대용량 데이터 / 안정적인 성능 / DB / 로그 처리(금융 시스템이나 로그 시간순 정렬, 주문 내역 날짜 순으로 정렬)

A) 퀵 정렬은 기준 값인 피벗을 중심으로 데이터를 분할하여 정렬하는 분할 정복 알고리즘이다.
->빠름 / 메모리 효율 / 배열 정렬 / 실시간(사용자 점수 랭킹순으로 정렬 등) 

A) 힙 정렬은 힙 자료구조를 이용하여 최대값 또는 최소값을 반복적으로 추출하여 정렬하는 알고리즘이다.
-> 우선순위 / 안정적인 성능 / 최대·최소값 추출(OS 스케줄러, 네트워크 패킷 처리)
```

- 각 정렬 방법의 시간 복잡도 및 적합한 사용 시나리오
```jsx
A) 버블, 선택, 삽입 정렬은 구현이 단순하지만 시간 복잡도가 높아 소규모 데이터에 적합하고 
병합, 퀵, 힙 정렬은 시간 복잡도가 낮아 대규모 데이터 처리에 적합하다.
```

## 3. 검색 알고리즘
- 선형 검색, 이진 검색
```jsx
A) 선형 검색은 데이터의 처음부터 끝까지 순차적으로 값을 비교하여 원하는 데이터를 찾는 알고리즘이다.
	데이터가 **정렬되어 있지 않거나 크기가 작을 때 사용된다**(명단 이름 찾기 등)

A) 이진 검색은 정렬된 데이터에서 중간 값을 기준으로 탐색 범위를 절반씩 줄여가며 검색하는 알고리즘이다.
	데이터가 반드시 정렬되어 있어야 한다.(사전에서 단어 찾기 등)
```

- 검색 알고리즘의 효울성 분석
```jsx
A) 검색 알고리즘의 효율성은 데이터의 크기 증가에 따른 탐색 시간의 변화로 분석한다.
	선형 검색은 O(n), 이진 검색은 O(log n)의 시간 복잡도를 가진다.
```

## 4. 자료 구조 활용과 알고리즘
- 스택과 큐의 활용
```jsx
A) 스택은 후입선출 구조로 함수 호출 처리에 활용되고 큐는 선입선출 구조로 작업 대기열 처리에 활용된다.
	스택은 되돌아가기 기능에, 큐는 순서 처리에 적합하다.
```

- 링크드 리스트 : 단일 링크드 리스트, 이중 링크드 리스트
```jsx
A) 단일 링크드 리스트는 다음 노드만 가리키고 
이중 링크드 리스트는 이전과 다음 노드를 모두 가리키는 구조이다.
	이중 링크드 리스트는 양방향 이동이 가능하지만 메모리 사용량이 더 크다.
```

- 트리와 그래프 : 이진 트리, 트리 순회 기법, 기본 그래프 알고리즘
```jsx
A) 이진 트리는 각 노드가 최대 두 개의 자식을 가지는 트리 구조이다.
	계층적인 데이터를 표현하는 데 적합하다.(폴더 구조)

A) 트리 순회 기법에는 전위 순회, 중위 순회, 후위 순회가 있으며 노드를 방문하는 순서에 따라 구분된다.
	중위 순회는 이진 탐색 트리에서 오름차순 결과를 얻을 수 있다

A) 기본 그래프 알고리즘에는 그래프 탐색을 위한 깊이 우선 탐색과 너비 우선 탐색이 있다.
	DFS(미로 탐색)는 깊이 탐색에, BFS(최단경로 찾기)는 최단 거리 탐색에 주로 사용된다.
```
